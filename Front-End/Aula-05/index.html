Envio de projetos ao git/github

Etapas:

  1. Editar/alterar/preparar os arquivos a serem enviados;
  2. Ao término das alterações, utilizar o git add para inclusão dos arquivos alterados no stage;
  3. Efetuar o commit das alterações com o comando git commit -m "mensagem", lembrando de inserir uma mensagem clara e objetiva listando as alterações efetuadas;
  4. Após o commit, o repositório local está atualizado. Em seguida, utilizamos o comando git push para enviar as alterações para o repositório remoto, caso ele exista;
  5. Repositório atualizado com sucesso! :)
